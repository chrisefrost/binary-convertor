## Understanding the Weird Terms

### Binary (Base 2)
- Only uses 0 and 1
- Each position is worth double the previous
- 1, 2, 4, 8, 16, 32, 64, 128...
- Like having only 2 fingers to count on!

### Hexadecimal (Hex, Base 16)
- Uses 0-9 and A-F (16 digits)
- Programmers use it because it's shorter than binary
- 4 binary digits = 1 hex digit
- Example: 1111 = F

### Octal (Base 8)
- Uses 0-7 (8 digits)
- Less common nowadays
- 3 binary digits = 1 octal digit

### ASCII
- A code that assigns a number to every letter, number, and symbol
- A=65, B=66, C=67, etc.
- a=97 (lowercase is +32 from uppercase)
- Space=32, !-=33, 0=48, etc.

### Two's Complement
- How computers handle negative numbers
- The first bit tells if it's negative
- If first bit is 1, the number is negative
- Example: 11111111 = -1 (in signed mode)

## Questions You Might Have

### "Why not just use regular numbers?"
Computers are made of electrical circuits. It's easier to detect ON vs OFF than 10 different voltage levels. Binary is simple and reliable!

### "Is hex really used by programmers?"
Yes! All the time. Memory addresses, color codes (#FF5733), file formats, network protocols, and more.

### "Why does 'a' = 97 and 'A' = 65?"
The ASCII table was designed that way. The difference is exactly 32, which makes programming easier (you can convert cases by flipping one bit!).

### "What's the point of learning this?"
- Helps you understand how technology works
- Useful for programming and computer science
- Makes you better at problem-solving
- It's actually kind of cool once you get it!
- Future careers: game dev, cybersecurity, AI, robotics
