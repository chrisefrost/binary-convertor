### üíª Code Contributions

#### Getting Started
1. Fork the repository
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages
6. Push to your fork
7. Open a Pull Request

#### Code Style
- Follow PEP 8 for Python code
- Add comments explaining complex logic
- Keep functions focused and simple
- Maintain separation between logic (converter_logic.py) and UI (binary_converter_app.py)

#### What We're Looking For
- **Educational value**: Does it help students learn?
- **Simplicity**: Is it easy to understand and use?
- **Reliability**: Does it work correctly in all cases?
- **Documentation**: Is it well-explained?

## First-Time Contributors

Never contributed to open source before? No problem! This is a great project to start with:

### Good First Issues
Look for issues labeled `good first issue` or `beginner-friendly`. These are specifically chosen to help newcomers get started.

### Easy Contributions
- Fix typos in documentation
- Add examples to FEATURES.md
- Improve error messages
- Add unit tests
- Improve comments in code

## Testing Your Changes

Before submitting:
1. Run the application: `python binary_converter_app.py`
2. Test all three tabs
3. Try edge cases (very large numbers, special characters, etc.)
4. Ensure no crashes or error messages

## Educational Focus

Remember, this is an educational tool! When contributing, ask yourself:
- Does this help students understand better?
- Is it intuitive for beginners?
- Does it support learning objectives?
- Is it explained clearly enough for a student?

## Questions?

Not sure about something? Ask! Open an issue with your question or propose your idea. We're here to help and learn together.
